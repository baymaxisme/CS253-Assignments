# CS253 Assignment 1: Memory-Efficient Versioned File Indexer


## Overview
This project implements a memory-efficient versioned file indexer in C++. It processes large text files incrementally using a fixed-size buffer to extract alphanumeric words and build a word-level index (mapping unique words to their frequencies). The program supports multiple versions and handles queries such as word counts, frequency differences between versions, and top-K most frequent words.
The system is designed strictly following Object-Oriented Programming (OOP) principles and guarantees that the entire file is never loaded into memory, ensuring memory usage scales only with the number of unique words, not the file size.


## Object-Oriented Design & C++ Requirements
The solution implements the following core components to satisfy the assignment requirements:

### 1. Classes and Responsibilities
`BufferedFileReader`: Manages file I/O operations. It reads large files in fixed-size chunks (between 256 KB and 1024 KB) using dynamic memory allocation to ensure strict memory limits are maintained.
`Tokenizer`: Parses raw text chunks into individual alphanumeric, case-insensitive words. It maintains a `leftover` state to correctly handle words split across buffer boundaries.
`VersionedIndexer`: The central data structure holding the word-level indices. It uses a nested `std::unordered_map` to store word frequencies mapped by version names.
`Query` (Abstract Base Class): Defines the interface for all queries.
`WordQuery`, `DiffQuery`, `TopKQuery`: Concrete classes implementing the `Query` interface for specific query types.

### 2. Required C++ Features Demonstrated
**Inheritance & Abstract Classes**: The `Query` class serves as an abstract base class with a pure virtual function `virtual void execute() = 0`;. `WordQuery`, `DiffQuery`, and `TopKQuery` inherit from it.
**Runtime Polymorphism**: Implemented in `main()` using a base class pointer (`Query* query`) that dynamically dispatches the `execute()` method based on the query type specified at runtime.
**Function Overloading**: Demonstrated in VersionedIndexer, which provides two versions of `addWord()`:`addWord(const std::string& version, const std::string& word)` and `addWord(const std::string& word) (defaults to a "default" version)`
`Exception Handling`: Utilized `try`, `catch`, and `throw`. E.g., throwing `std::runtime_error` for invalid buffer sizes or failed file openings, and catching them gracefully in `main()`.
Templates: Utilized a function template `template<typename T>` void printVector(`const std::vector<std::pair<T,int>>& vec`) to generically print vector contents.

## Usage and Examples
The program processes arguments via the command line.

### Command-Line Arguments:
`--file <path>`: Path to input file
`--file1 <path>` / `--file2 <path>`: Paths for difference queries
`--version <name>`: Version identifier
`--version1 <name> `/ `--version2 <name>`: Identifiers for difference queries
`--buffer <kb>`: Buffer size in KB (must be 256 - 1024)
`--query <type>`: Type of query (word, diff, top)
`--word <token>`: Target word for word and diff queries
`--top <k>`: Integer k for the top query
